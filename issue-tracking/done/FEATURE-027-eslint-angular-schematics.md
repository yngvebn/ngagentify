# Run ng add @angular-eslint/schematics in demo

## Summary

Use the Angular CLI schematic to wire ESLint into the demo Angular project. This updates `angular.json`, generates `eslint.config.mjs` for the demo, and installs `@angular-eslint/eslint-plugin`.

## Acceptance Criteria

- Run from `demo/` directory: `ng add @angular-eslint/schematics`
- `demo/angular.json` has a `lint` target added to `ng-annotate-demo` project:
  ```json
  "lint": {
    "builder": "@angular-eslint/builder:lint",
    "options": { "lintFilePatterns": ["src/**/*.ts", "src/**/*.html"] }
  }
  ```
- `demo/eslint.config.mjs` is generated by the schematic and references `@angular-eslint`
- `demo/eslint.config.mjs` is updated to also extend the root config (add `...rootConfig` at the top)
- `ng lint --project=ng-annotate-demo` exits 0 on the scaffolded demo app
- `@angular-eslint/eslint-plugin` and `@angular-eslint/template-parser` are in demo's devDependencies

## Technical Notes

- Run `ng add` inside the `demo/` directory, not from the repo root, or use `--project=ng-annotate-demo` flag
- After the schematic runs, manually add the `extends rootConfig` reference to the generated `demo/eslint.config.mjs`
- Never configure Angular ESLint by hand when the schematic can do it (per spec policy 0.7)
